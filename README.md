# odoo_slavakulikov

Модуль разработан для **Odoo Community Edition 18** и расширяет модель `sale.order`.

## Установка и запуск проекта

Клонировать репозиторий и перейти в директорию проекта:

```bash
git clone https://github.com/SlavaKlkv/odoo_slavakulikov.git
cd odoo_slavakulikov
```

Запустить контейнеры:

```bash
docker compose up
```

После запуска Odoo будет доступен по адресу:

```text
http://localhost:8069
```

## Установка модуля

1. Открыть Odoo в браузере.
2. Перейти в **Apps**.
3. Убрать фильтр **Apps**
4. Найти модуль **Odoo Slava Kulikov**.
5. Нажать **Activate / Install**.

## Проверка функционала

### 1. Проверка формы Quotation

Путь:

```text
Sales > Quotations > New
```

Проверить, что:

- появилось поле **Ответственный за выдачу товара**;
- в поле можно выбрать сотрудника из раздела **Employees (`hr.employee`)**;
- поле обязательное;
- в `New Field` подставляется случайное значение длиной 10 символов;
- при изменении даты (**Quotation Date**) или строк заказа `New Field` меняется на значение в формате `Date + Total`.

### 2. Проверка статусов

Для `New Field`:

- `Draft` → поле редактируемое;
- `Sent` → поле только для чтения;
- `Sale` → поле скрыто.

### 3. Проверка ограничения длины

- вручную ввести в `New Field` строку длиной более 30 символов и кликнуть в любом другом месте;
- убедиться, что показывается предупреждение (при изменении значения поля);
- при сохранении запись не должна сохраняться, должно появиться сообщение:

```text
Длина текста должна быть меньше 30 символов!
```

### 4. Проверка печатной формы

Путь:

```text
⚙️ в левом верхнем углу > Print > Quotation / Order
```

Проверить:

- если `New Field` заполнено — значение выводится над таблицей товаров;
- если `New Field` пустое — выводится:

```text
Поле New Field пустое!!!
```

## Примечания

- проект проверялся локально через Docker Compose;
- модуль рассчитан на установку в чистый экземпляр Odoo Community 18.
